# 交互设计规范 (NEW)

## 核心理念
- **去掉 vim 模式** - 不再使用 j/k/h/l 导航
- **方向键优先** - 使用标准方向键
- **直观操作** - 符合现代编辑器习惯
- **无模式切换** - 简化操作逻辑

## 快捷键设计

### 导航
| 按键 | 动作 |
|------|------|
| `↑/↓` | 上/下移动 |
| `←/→` | 折叠/展开（若有子节点）或 返回父节点 |
| `Enter` | 展开节点并进入 / 确认操作 |
| `Home/End` | 跳到首行/末行 |
| `PageUp/PageDown` | 向上/下翻页 |

### 编辑
| 按键 | 动作 |
|------|------|
| `Enter` | 编辑当前值（叶子节点） |
| `Escape` | 取消编辑 |
| `Delete` | 删除当前节点 |

### 增删
| 按键 | 动作 |
|------|------|
| `Insert` 或 `N` | 添加新节点 |
| `Delete` | 删除节点 |
| `Ctrl+C` | 复制值 |
| `Ctrl+V` | 粘贴（todo） |

### 通用
| 按键 | 动作 |
|------|------|
| `Ctrl+S` | 保存 |
| `Ctrl+F` | 搜索 |
| `Ctrl+Z` | 撤销 |
| `Ctrl+Y` | 重做 |
| `F2` | 右键菜单 |
| `Space` | 切换展开/折叠 |
| `Ctrl+Q` | 退出（未保存提示） |
| `F1` | 帮助 |

### 搜索模式
| 按键 | 动作 |
|------|------|
| `Enter` | 跳到下一匹配 |
| `Escape` | 退出搜索 |
| `↑/↓` | 上一/下一匹配 |

### 添加节点模式
| 按键 | 动作 |
|------|------|
| `Tab` | 切换 key/value 焦点 |
| `Enter` | 确认添加 |
| `Escape` | 取消 |

## 设计原则
1. **易学** - 快捷键符合常见编辑器习惯
2. **一致性** - 相似的操作使用相似的按键
3. **最小化** - 只保留最必要的快捷键
4. **无模式** - 尽量减少模式切换
